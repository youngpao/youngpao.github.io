## 高级事件绑定 ##

addEventListener

注意细节：

1.事件名：一律不带on

2.绑定事件函数中的“this” 指绑定该事件的对象

3.执行顺序，是按绑定顺序来执行

    <!DOCTYPE html>
    <html>
    <head>
    	<title></title>
    </head>
    <body>
    <input type="button" value="dian">
    </body>
    <script type="text/javascript">
    	var bt=document.getElementsByTagName('input')[0];
    	bt.addEventListener('click',function (){alert('haha')});
    	bt.addEventListener('click',function (){alert('hehe')});
    </script>
    </html>

这样连续点两下的值都能出现

如果要删除指定的事件必须要把函数变成值

    <!DOCTYPE html>
    <html>
    <head>
    	<title></title>
    </head>
    <body>
    <input type="button" value="dian">
    <input type="button" value="del">
    </body>
    <script type="text/javascript">
    	var bt=document.getElementsByTagName('input')[0];
    	var haha=function (){alert('haha')};
    	bt.addEventListener('click',haha);
    	bt.addEventListener('click',function (){alert('hehe')});
    
    	//移除事件
    	var bt1=document.getElementsByTagName('input')[1];
    	bt1.onclick=function (){
    		bt.removeEventListener('click',haha);
    	}
    </script>
    </html>

## 停止传播 ##

事件对象有一个方法可以阻止事件的传播：stopPropagation()

	//阻止捕捉
	d2.addEventListener('click',function(ev){alert('d2');ev.stopPropagation();},true);

	//阻止冒泡
	d2.addEventListener('click',function(){alert('md2');ev.stopPropagation();},false); 


## 作用域 ##

js 函数内部 找变量会优先找函数内部，然后一层层往上找，直到函数内部没有，那么找外部全局变量

作用域测试

    <script type="text/javascript">
    	var str='global';
    
    	function t(){
    		console.log(str); //global
    		console.log(str1); //undefined
    		var str1='local';
    	}
    
    	t();
    </script>

为什么str1是undefined 不是 local呢

这是因为 js执行前有一个词法分析过程

分析过程：

1.先分析参数

2.再分析变量声明

3.分析函数声明

一个函数能使用的局部变量就从上面三步来

具体步骤：

1.函数运行钱的一瞬间，生成active object 简称AO

2.函数声明的参数形成AO 的属性，值为实参的值，未传实参则为undefined

3.分析变量声明 如 var age 

如果AO上还没有age 属性，则添加AO属性，但是undefined

如果AO上已经有age属性，则不做任何影响

4.分析函数声明，如 function foo(){}

则把函数献给AO.foo属性

注意，如果此前foo属性已存在，则被覆盖掉

    <script type="text/javascript">
    	var age=99;
    	function t(age){
    		console.log(age);
    	}
    	t(5); //5
    	
    </script>

词法分析过程：

1.函数运行钱的一瞬间，生成active object 简称AO

分析结果 t.AO={}

2.函数声明的参数形成AO 的属性，值为实参的值，未传实参则为undefined

分析结果 t.AO={age=5}

3.分析变量声明 如 var age 

如果AO上还没有age 属性，则添加AO属性，但是undefined

如果AO上已经有age属性，则不做任何影响

结果：没有，略过

    <script type="text/javascript">
    	var age=99;
    	function t(age){
    		console.log(age);
    	}
    	t(5); //5
    	t(); //undefined
    
    </script>

同样代码，第二个结果就是undefined

    <script type="text/javascript">
    	
    	function t(age){
    		var age=5;
    		console.log(age);
    	}
    	
    	t(99); //5
    </script>


按照分析过程 age=99

但是最后还有一步执行，执行之后age= 5


## var 是声明变量 ##

var 是在函数运行的上下文中声明一个变量

如果不加var，，那就是赋值操作

    <script type="text/javascript">
    	a=1;
    	function t(){
    		var b=3;
    		a=2;
    		console.log(b+a); 
    	}
    
    	t();//5
    	console.log(a); //2
    
    </script>

这样打印全局变量a 就变成函数变量

    <script type="text/javascript">
    	a=1;
    	function t(){
    		var b=3;
    		var a=2;
    		console.log(b+a); 
    	}
    
    	t();//5
    	console.log(a); //1
    
    </script>

加上var 那么全局变量a就不会受到函数影响

这样就可以各归各的使用

## 函数声明与函数表达式 ##

js中函数也是一种值

可以复制给变量，可以作为参数来传递

function t1(){}

t1= function(){}

这两种方式效果不同的

第一个是函数声明，虽然全局内也得到一个t1变量，值是function

第二个t1 只是一个赋值过程，值是右侧表达式的返回结果

因此 这两种方式在词法分析时有着本质区别

前者在词法分析阶段就发挥作用

后者在运行阶段才发挥作用

##arguments对象 ##

函数运行其内，关键的三个对象

AO:本函数AO上没有某属性，则继续去外层和桉树AO上找，叫做作用域链

arguments:每个函数有自己的arguments，但不向外层接着找arguments 的相关属性，即 不形成链

this: 沿原型查找，形成原型链

arguments 是一个对象，是一个长的很像数组的对象

arguments 的内容是函数调用时的实参列表

    <script type="text/javascript">
    function sum(){
    	console.log(arguments);
    }
    
    sum(1);
    sum(1,2);
    sum(1,2,3);
    </script>

![](http://i.imgur.com/RXh5Jo9.png)


    <script type="text/javascript">
    function sum(){
    	for(var sum=0,i=0;i<arguments.length;i++){
    		sum+=arguments[i];
    	}
    
    	console.log(sum);
    }
    
    sum(1); //1
    sum(1,2); //3
    sum(1,2,3); //6
    </script>


## callee属性 ##

callee就代表函数自身

    <script type="text/javascript">
    var r=(function (n){
    	if (n>1) {
    		return n+arguments.callee(n-1);
    	}else{
    		return 1;
    	}
    })(100);
    
    console.log(r); //5050
    
    </script>

## this ##

js的this很灵活，可以按照函数的4种调用方式分别讨论

1.作为普通函数来调用时

this的值指向->window

准确的说，this为null，但被解释成window

    <script type="text/javascript">
    function t(){
    	this.age=23;
    }
    t(); //调用者为null
    
    alert(window.age); //23
    </script>


现在当this为null时，会抛出异常

2.哪个对象调用的，this就指向谁

    function intro(){
    	alert('my name is ' +this.name);
    }
    
    var you={name:"young"};
    you.intro=intro; //my name is young

3.作为构造函数来调用时，this指向该空对象

     function cat(name,color){
     	this.name=name;
     	this.color=color;
     }
    
     new cat(); //会得到一个新对象,方法内的this指向该对象
